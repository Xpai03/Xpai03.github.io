实验概述

任务：要求编写两个图片处理的函数：将图片逆时针旋转90°、利用平均值替换的方式使图片平滑。并进行程序性能优化，最后根据函数运行的 CPE 得到分数。

在做过ArchLab、cacheLab后对程序性能优化有了大致的了解，合理利用缓存、循环展开、矩阵分块、减少内存引用，减少不必要的函数调用等技术来优化程序运行的时间。

分块技术

    - 根据**假设图像维度为32的倍数**这一个条件可以试着将该图像划分成几个32*32或8*8的矩阵，分块进行操作
    - 为了充分利用一级缓存（32kB），故采用分块策略，将每一小块分成32*32或8*8的矩阵
    - 这里的分块是指一个应用级的数据组块，而不是高速缓存中的块，这样构造程序，能将一个片加载到L1高速缓存中去，并在这个片中进行所需要的所有读和写，然后丢掉这个片，加载下一个片，以此类推。

使用分块技术，将矩阵分成相同规格的小矩阵，对这些小矩阵进行rotate操作，在这里测试了88、3232分块，并比较性能，发现8*8效果更好。

分析原因可能是：

  • 较小的块大小（如8x8）在性能上可能更有优势，因为它们有助于利用数据的局部性，并减少内存访问的频率。

  • 大块的循环展开可以提供更多的并行性，但也可能导致更多的缓存未命中。这可能会导致性能下降，因为需要从主存中频繁加载数据，而这会引入更多的延迟。

循环展开

根据之前从ArchLab中学到的循环展开方法，将外层和内层循环都使用循环展开，每次迭代处理 8 行或 8 列像素数据，以此来减少循环迭代的次数。

最终优化

  • 最外层的两个循环 for (int i = 0; i < dim; i += 8) 和 for (int j = 0; j < dim; j += 8) 用于将图像划分为 8x8 的块，以提高数据访问的局部性，从而提高性能。

  • 内循环 for (int k = i; k < i+8; k++) 用于遍历每个 8x8 的块。

  • 在内循环中，针对每个像素位置，将原始图像中的像素值复制到目标图像中，但是有一个变换，即 dst 中的行和列的索引值需要调整，以实现逆时针旋转。这是通过 RIDX 宏来实现的，它将原始图像和目标图像的行和列索引值重新映射，以实现旋转。
